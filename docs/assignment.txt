I. OVERVIEW
In this assignment you will work with the FLNET2023 dataset [1], a realistic network intrusion detection dataset originally
designed for federated learning research. The dataset was generated by emulating Europe’s GEANT-2012 network topology
(40 nodes) using the Common Open Research Emulator (CORE) [2]. Network traffic was collected at ten strategically placed
data-collecting routers (D1,D2,...,D10) and processed through CICFlowMeter [3] to extract 82 flow-based features per
flow record.
The original dataset contains 11 traffic classes: one benign (Normal) and ten attack categories spanning DDoS, DoS, Web
attacks, and Infiltration. You will receive the dataset with all class labels removed. Your task is to analyze the unlabeled
traffic, reconstruct the network topology, discover and classify attacks, and train high-performance machine learning models –
all without access to ground-truth labels during the discovery phase.
A. Dataset Access
The unlabeled dataset files can be downloaded from FLENT Dataset .Each file corresponds to one data-collecting router
(Di). Each row is a single network flow with 82 CICFlowMeter features plus a router_id column identifying the collecting
router. The label column has been removed.
Reference repository (for documentation only): https://github.com/nsol-nmsu/FML-Network
B. Dataset Scale
For reference, the combined dataset (train + test) contains approximately 7.5 million flow records distributed unevenly
across the ten routers. The class distribution is highly imbalanced – ranging from hundreds of instances for rare attack types
to millions for dominant classes.

VIII. HINTS AND TIPS
Tip 1: Start with Exploratory Data Analysis . Before diving into clustering or modeling, spend substantial time under-
standing the data. Plot distributions, check for anomalies, understand what each CICFlowMeter feature represents. The
CICFlowMeter documentation1 describes all 82 features.
Tip 2: Memory management matters. With 7.5M rows and 82+ columns, the full dataset may not fit in memory on all
machines. Consider: reading data in chunks, using float32 instead of float64, using polars instead of pandas
for faster processing, or working with stratified subsamples for exploratory analysis.
Tip 3: Think about attack signatures. Different attacks have distinct traffic fingerprints:
• DDoS attacks typically involve high packet rates and many flows to the same destination.
• DoS Hulk generates many HTTP requests; DoS SlowHTTP holds connections open with minimal data.
• Web attacks (SQLi, XSS, Command Injection) tend to be low-volume with unusual payload sizes.
• MITM (Infiltration) involves subtle traffic interception patterns.
Tip 4: For topology reconstruction, remember that routers that share traffic to common destinations or exhibit correlated
flow timing are likely topologically close. The GEANT-2012 topology has a known structure – use your networking
knowledge about ISP backbone topologies to guide your reconstruction.
Tip 5: For class imbalance, note that naive SMOTE on 6.8M samples is computationally prohibitive. Consider: applying
SMOTE only to minority classes, using ADASYN which focuses on hard-to-learn samples, or using algorithmic
approaches (focal loss, class weights) which require no resampling.
Tip 6: Evaluation beyond accuracy. Given the severe imbalance, accuracy alone is misleading. A model predicting “Normal”
or “DDoS TCP” for everything could achieve high accuracy. Always report macro F1-score and per-class metrics.
Tip 7: Reproducibility. Set random seeds for all experiments. Document your environment. The graders will spot-check code
execution.
